🧪 기회 — 무엇을 만들까

  • meta-harness 스타일 최소 오케스트레이션 구축: planner 1 + scoped subagent 2-3개를 별도 브랜치로 병렬 실행 후 merge/review, 단일 에이전트 순차 실행 대비 token 비용·conflict rate 측정.
  • Claude Code로 작은 PyTorch 모델을 ONNX Runtime Web + WebGPU 클라이언트 사이드로 end-to-end 포팅하는 browser ML 데모(Moebius 패턴).
  • 에이전트 행위에 대한 책임/감사를 위한 production observability + online eval 파이프라인(저비용 trace 이슈 탐지 모델 포함).
  • 'MCP를 auth gateway로 축소'하는 thin passthrough 패턴 vs full-featured MCP 서버의 성공률·latency 비교 도구화.

⚠️ 리스크 — 무엇을 경계할까

  • 프런티어 모델 접근이 정부 매개·risk-tiered 통제로 이동할 가능성 — 단일 모델·단일 벤더 종속 회피 필요.
  • multi-agent disagreement loop 등 통제되지 않은 자율성이 runaway inference 비용/되돌릴 수 없는 손상을 유발(코드 리뷰 무한 루프 사례).
  • prompt injection·role confusion: 6,000회 방어 성공이 미래 공격 안전을 보장하지 않음 — 비가역 손상 가능 지점엔 에이전트 배포 금지.
  • supply chain 공격(unpinned dependency, npm/pip)으로 자격증명 유출 — 'AI가 다 했다'식 책임 회피 인센티브 경계.
  • LLM의 단방향 설득력·sycophancy, 그리고 모델 내부 감정 귀속 같은 과대 해석에 휩쓸리지 말 것.
