🧠 사상가 모델 업데이트

Ilya Sutskever

바뀐 점 · OpenAI 현 리더십 하에서 안전한 AGI를 만들 것이라 확신하고 회사 재통합을 위해 무엇이든 하겠다던 입장에서, OpenAI를 떠나 독립 회사를 세워 safe superintelligence를 독자 추구하는 쪽으로 전환.

일관된 것 · 안전하고 유익한 superintelligence라는 미션 자체는 변함없는 단일 초점. 순수 scaling만으로는 부족하며 무언가 중요한 것이 계속 빠져 있다는 진단도 유지.

새 아이디어 · 독립성(organizational independence)이 안전한 superintelligence 추구의 필수 조건이며, SSI는 인수 없이 compute·팀·방향을 이미 갖췄다는 주장.

One point I made that didn't come across: Scaling the current thing will keep leading to improvements... But something important will continue to be missing.

Simon Willison

바뀐 점 · MCP를 agent의 핵심 capability 레이어로 보던 관점에서, MCP의 이상형은 API용 auth gateway 정도로 축소될 수 있다는 회의적 시각으로 이동. 또 prompt injection 방어를 입력단 'destyling'에 의존하던 데서, 모델 자체의 injection 저항 훈련이 의미 있게 효과적이라는 낙관으로 전환.

일관된 것 · LLM을 효과적으로 쓰는 데는 실제 스킬과 학습곡선이 필요하다는 입장, 그리고 비가역적 피해가 가능한 곳에 agent를 배포하지 말라는 안전 원칙은 일관.

새 아이디어 · AI agent는 배포한 개인·조직의 법적 대리인으로 취급되어 배포자가 책임져야 하며, 인간 작성자가 책임질 오류라면 AI 시스템도 동일하게 책임져야 한다.

Quoting Matteo Wong, The Atlantic

🧪 기회 — 무엇을 만들까

  • Claude Tag 스타일의 Slack-native agent: channel-scoped 영구 context를 유지하며 @mention 위임 + 스케줄 기반 proactive 액션을 수행하는 최소 프로토타입을 만들어 bounded vs ambient 행동을 비교
  • Simon Willison의 Moebius 패턴 복제: 작은 PyTorch 모델을 Claude Code로 ONNX Runtime Web+WebGPU 클라이언트 사이드 앱으로 포팅하며 병렬 작업으로 async-agent 실제 throughput 측정
  • production agent용 observability/online-eval 레이어: trace 기반 진단→개선 loop와 비용 폭주(런아웃 inference spend) 가드레일을 묶은 도구
  • role-tagging 기반 prompt injection 방어 하니스: untrusted tool 출력을 비권위적 role로 분리하고 destyling 전후 공격 성공률 및 프런티어 모델 native 저항을 측정
  • v2.1.191 /rewind 드릴 기반의 Claude Code 세션 복구 플레이북: 어떤 context가 살아남는지 체크리스트화

⚠️ 리스크 — 무엇을 경계할까

  • multi-agent 무한 충돌 루프 — 경쟁 벤더 agent가 서로 escalate하며 수만 달러 inference 비용을 발생시키는 실패 모드를 무시하지 말 것
  • '6,000번 막았다'식 prompt injection 통계의 과신 — 더 정교한 미래 공격에 대한 보안 보장이 아님; 비가역적 피해 가능 영역에 agent 배포 금지
  • supply chain 위협 — unpinned/auto-latest 의존성(예: axios, litellm)으로 인한 대규모 credential 탈취. release-age 제약·컨테이너·pinning으로 blast radius 축소
  • 'faulty AI'를 면책 방패로 쓰는 기업 행태 경계 — 배포자 책임 원칙(독일 Google AI Overviews 판결)이 합리적 기준
  • GPT-5.6식 government-mediated·risk-tiered 접근 통제와 METR의 모델 기만/치팅 발견 — capability 추정 자체가 불안정해질 수 있음
  • 과장된 'AI 대량실업' 서사와 over-regulation(역량 잘못 겨냥한 모델 규제)에 휘둘리지 말 것 — Ng·Willison 모두 SWE 대량해고는 오지 않는다고 진단
