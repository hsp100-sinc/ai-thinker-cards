🧪 기회 — 무엇을 만들까

  • 동일 코딩 태스크를 Codex CLI와 Claude Code에 돌려 토큰·시간·tool call·first-try CI 통과를 로깅하고 'useful work per dollar'로 채점하는 벤치마크 하네스
  • 단일 모델을 감싸는 최소 harness(state file + invariant check + retry + commit 전 verification)를 만들어 10-task 배치에서 one-shot 대비 신뢰성 비교
  • conformance test suite로 현재 동작을 고정한 뒤 병렬 subagent(git worktree)로 소형 모듈을 다른 언어/프레임워크로 재작성 — Bun-in-Rust식 재작성 패턴 검증
  • sqlite-utils 기반 공유 DB에 findings를 쓰는 병렬 security-sweep 하네스(디렉토리 슬라이스별 스캔 + 통합·랭킹 에이전트)로 대규모 병렬 오케스트레이션 실습
  • production agent trace에서 실패 패턴을 탐지하는 저렴한 post-trained 소형 judge 모델 + online eval 파이프라인
  • memory 파일 표준(BYOAI)으로 Claude/Codex/OpenCode 어떤 agent든 같은 개인 지식베이스에 접속하게 하는 도구

⚠️ 리스크 — 무엇을 경계할까

  • forward-dated/speculative 이슈 다수 — GPT-5.6 Sol, Claude Fable 5, Kimi K3 등 미래·가상 시나리오성 model명과 수치는 확정 사실이 아닌 프로젝션으로 취급
  • permission model·data privacy 무시 금물: Grok CLI의 기본 디렉토리 업로드, Claude web_fetch의 링크 추적 exfiltration 등 agent tool의 lethal trifecta 취약점
  • runaway subagent·web search 루프로 인한 비용·자원 폭주 — hard cap 없이 advisory에만 의존하지 말 것
  • 'better model, worse tool' 회귀 — 강한 모델일수록 custom tool schema 준수가 나빠질 수 있어 harness의 edit tool 다중 구현 필요
  • auto mode로 리뷰 없이 production 코드를 방치하는 유혹 — coding agent가 synchronizing friction을 제거해 프로젝트 공유 이해가 침식될 위험
  • token leaderboard가 부추기는 성과 과시성·낭비적 AI 사용과 벤더-고객 간 '100x 생산성' 과장 피드백 루프
